========================================================================
SUMMARY OF ASSESSMENT ROUND FIXES & UPDATES
========================================================================

1. BACKEND - ASSESSMENT AI SERVICE (`Backend/assessments/ai_service.py`)
------------------------------------------------------------------------
- Updated `generate_assessment_questions()` to generate a pure, generic Software Engineering Assessment Test.
- Strictly prohibited any resume-based, project-specific, or candidate-specific questions in the prompt.
- Set category structure to EXACTLY 35 questions across 7 core domains (5 questions each):
  * Verbal Ability (5 questions)
  * Logical Reasoning (5 questions)
  * Operating System (5 questions)
  * Computer Network (5 questions)
  * DBMS (5 questions)
  * OOPs (5 questions)
  * Data Structures & Algorithms (5 questions)
- Added automatic fallback filling: If the AI API returns fewer than 35 questions (or fails parsing), the function dynamically fills missing slots from a complete 35-question static fallback pool, guaranteeing EXACTLY 35 MCQs every time.

2. BACKEND - ASSESSMENT VIEWS (`Backend/assessments/views.py`)
------------------------------------------------------------------------
- Updated `get_questions` endpoint to check if existing questions for an assessment round have at least 35 questions.
- If a user previously had an assessment round with legacy 10-question data, `get_questions` automatically purges the old 10-question data from the database and replaces it with the full 35-question set.
- Removed all resume/role fallback parameters when fetching assessment questions.

3. BACKEND - INTERVIEW CREATION (`Backend/interviews/views.py`)
------------------------------------------------------------------------
- Updated `InterviewCreateView.perform_create` to pre-generate and save the 35 generic assessment questions upon assessment session creation, clearing any previous questions linked to the interview instance.

4. FRONTEND - ASSESSMENT UI (`Frontend/src/pages/Assesment.jsx`)
------------------------------------------------------------------------
- Increased assessment test timer from 15 minutes (900s) to 30 minutes (1800s) to comfortably accommodate 35 questions.
- Added visual category tags (e.g. "VERBAL ABILITY", "LOGICAL REASONING", "OPERATING SYSTEM", "COMPUTER NETWORK", "DBMS", "OOPS", "DSA") above each question card.
